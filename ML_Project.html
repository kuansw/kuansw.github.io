Using Machine Learning on HAR Sensors to Predict Activity Execution Correctness

Kuan Siew Weng

11 June 2015

Synopsis

Many people now regularly use personal activity tracking devices such as Jawbone Up, Nike FuelBand, and Fitbit to quantify how much of a particular activity they do, but these devices rarely quantify how well they do those activities.

The goal of this project is to determine how effective are the different machine learning predictive models in predicting “how well” a particular activity was performed, from the motion sensor data collected from individuals.

Source Dataset

The dataset used for this project is the Weight Lifting Exercise Dataset, which contains sensor measurements taken from six young healthy participants as they performed one set of 10 repetitions of the Unilateral Dumbbell Biceps Curl in five different fashions:

  • Class A : performed exactly according to the specification
  • Class B : throwing the elbows to the front
  • Class C : lifting the dumbbell only halfway
  • Class D : lowering the dumbbell only halfway
  • Class E : throwing the hips to the front

More information on this study and the dataset are available from the website : http://groupware.les.inf.puc-rio.br/har and in their following reference paper.

Velloso, E.; Bulling, A.; Gellersen, H.; Ugulino, W.; Fuks, H. Qualitative Activity Recognition of Weight Lifting Exercises. Proceedings of 4th International Conference in Cooperation with SIGCHI (Augmented Human ’13) . Stuttgart, Germany: ACM SIGCHI, 2013.

Data Exploration

We start with loading the training dataset and a quick exploratory scan of the loaded raw training dataset showed that the dataset comprised of 19622 records of 160 columns. It is observed that there are two types of records in the dataset:

  • observation records, which records the data from the sensors for the activity performed.
  • summary records, which aggregates the measurements into the summary statistics columns for a given time window. Summary records are indicated by the value of the column new_window is “yes”

Data Cleansing and Preparation

Since the goal of this project is to predict the value of classe from the observed sensor data, summary records shall be removed. Their summary statistic columns have NA values in observation records and shall be removed as well.

Finally, the first seven columns including timestamps, X and user_name are dimensional and are not needed in our prediction model. They shall also be removed from the training dataset.

After data cleansing, the number of records is 19216 and the cleansed record comprise of the response variable classe, and 52 predictor variables.

# remove all summary records indicated by new_window=yes
train_clean <- train_raw[train_raw$new_window == "no", ]

# remove summary stats columns which have NA values in remaining observation records
train_clean <- train_clean[,colSums(is.na(train_clean)) != nrow(train_clean)]

# remove dimension columns not used in prediction model
train_clean <- train_clean[,-c(1:7)]
dim(train_clean)
## [1] 19216    53

Partitioning Training Dataset for Cross-Validation

For the purpose of measuring and comparing the prediction accuracy of our prediction models, 30% of the cleansed training dataset shall now be set aside for use as the validation dataset.

# split train_clean to train_ds and val_ds for cross validation 
in_train <- createDataPartition(y=train_clean$classe, p=0.70, list=FALSE)
train_ds <- train_clean[in_train,]
val_ds <- train_clean[-in_train,]

Prediction Model 2 - Random Forest Model Using All Predictors

set.seed(33833)
no_of_folds <- 7

# Setting up the seeds for TrainControl to ensure reproducibility 
seeds <- vector(mode = "list", length = no_of_folds)
for(i in 1:no_of_folds) seeds[[i]] <- sample.int(1000, 3)
seeds[[no_of_folds + 1]] <- sample.int(1000, 1)

trc <- trainControl(method="cv", number=no_of_folds, seeds=seeds)

rf_fit <- train(classe ~ ., method="rf", data=train_ds, ntrees=150, prox=FALSE,
                trControl=trc)
